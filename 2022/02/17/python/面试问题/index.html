面试问题

  1. 什么是生成器？生成器和普通函数有何不同？
    回答：

生成器是使用 yield 关键字的函数，会返回一个迭代器，调用它时不会立即执行，而是返回一个生成器对象。
普通函数通过 return 一次性返回结果并结束执行，而生成器可以通过 yield 多次生成值，且每次调用时能恢复状态继续执行。

生成器和列表解析的区别是什么？
回答：

生成器表达式用 () 定义，而 列表解析用 []。
列表解析一次性生成所有元素，消耗较多内存，而生成器表达式按需生成元素，内存开销小。
示例：
python
复制代码
gen = (x * x for x in range(1000)) # 生成器
lst = [x * x for x in range(1000)] # 列表

如何中断生成器？生成器终止后会发生什么？
回答：

生成器在以下情况会终止：
yield 执行完毕后没有后续代码。
主动抛出 StopIteration 异常。
外部调用 gen.close()。
生成器终止后，试图继续获取值会抛出 StopIteration 异常。

yield 和 return 的区别是什么？
回答：

yield 用于生成器函数，会暂停执行并返回一个值，而不会终止函数。
return 用于普通函数，返回值后函数执行结束。如果在生成器中使用 return，会导致抛出 StopIteration 异常。

生成器能否进行双向通信？
回答： 是的，生成器支持双向通信。通过 generator.send(value) 方法可以向生成器中传值。send 的值会成为 yield 表达式的返回值：

python
复制代码
def generator():
value = yield “Start”
yield value

gen = generator()
print(next(gen)) # 输出 “Start”
print(gen.send(“Data”)) # 输出 “Data”

生成器在测试中有什么应用场景？
回答：

数据流模拟：生成器可以模拟大规模或动态变化的数据流。
在测试开发中，我们可能需要模拟从外部服务（如 API、数据库、文件系统等）连续流入的大量数据。如果一次性将所有数据加载到内存中，会导致资源消耗过大，甚至内存溢出。这时可以用生成器按需生成模拟数据。
懒加载数据：测试中生成动态数据以避免提前加载。
在一些测试场景中，不需要立即获取所有数据，而是根据需要逐步加载。例如：
分页加载大量数据。
从数据库或文件中逐行读取。
测试需要实时处理的数据流。
生成器可以用于懒加载，在需要时才计算或加载下一批数据。
生成器逐行读取文件内容，而不是一次性将整个文件加载到内存。
避免了内存占用过高的问题，尤其适合处理超大文件或实时数据。

适用的可迭代对象
for 循环可以用于任何实现了 迭代协议 或 序列协议 的对象：

迭代协议：实现了 iter() 方法并返回一个迭代器。
序列协议：实现了 getitem() 方法，从索引 0 开始按顺序获取元素，直到抛出 IndexError。

Python 中生成器的内置方法是生成器对象提供的用于交互和控制生成器的操作接口。这些方法包括：

  1. next()
    获取生成器的下一个值。
    每次调用时，生成器运行到下一个 yield 表达式，暂停并返回值。
    如果没有更多值，抛出 StopIteration 异常。
    示例：

python
复制代码
def my_gen():
yield 1
yield 2
yield 3

gen = my_gen()
print(gen.next()) # 输出: 1
print(gen.next()) # 输出: 2
print(gen.next()) # 输出: 3

print(gen.next()) # 抛出 StopIteration

  1. send(value)
    恢复生成器的执行，并将一个值发送到生成器中作为暂停点的 yield 表达式的结果。
    第一次调用生成器必须传入 None，因为生成器尚未运行到 yield。
    用于与生成器交互，实现动态数据传递。
    示例：

python
复制代码
def my_gen():
x = yield “Start”
yield f”Received: {x}”

gen = my_gen()
print(next(gen)) # 输出: Start
print(gen.send(“Hello”)) # 输出: Received: Hello

print(gen.send(“World”)) # 抛出 StopIteration

  1. throw(type, value=None, traceback=None)
    在生成器暂停的地方抛出一个指定类型的异常。
    如果生成器内部捕获了异常，生成器可以继续运行；否则，异常会传播到调用方。
    用于测试异常处理逻辑或中断生成器。
    示例：

python
复制代码
def my_gen():
try:
yield 1
yield 2
except ValueError as e:
yield f”Exception caught: {e}”
yield 3

gen = my_gen()
print(next(gen)) # 输出: 1
print(gen.throw(ValueError, “Error!”)) # 输出: Exception caught: Error!
print(next(gen)) # 输出: 3
4. close()
关闭生成器，使其终止执行。
在生成器暂停的地方抛出 GeneratorExit 异常，触发清理逻辑。
如果生成器捕获了 GeneratorExit，可以执行相关清理代码；否则生成器会直接结束。
示例：

python
复制代码
def my_gen():
try:
yield 1
yield 2
finally:
print(“Generator is closing”)

gen = my_gen()
print(next(gen)) # 输出: 1
gen.close() # 输出: Generator is closing

print(next(gen)) # 抛出 StopIteration

生成器内置方法总结
方法 功能描述 典型用途
next() 获取生成器的下一个值；运行到下一个 yield 表达式并暂停。 遍历生成器中的值。
send(value) 向生成器发送一个值作为上次 yield 表达式的结果，并继续执行。 实现生成器和调用方之间的双向数据通信。
throw(type) 在生成器暂停的地方抛出异常；可选择捕获或传播。 测试生成器的异常处理能力，或手动中断生成器。
close() 关闭生成器，在暂停点触发 GeneratorExit 异常。 提前终止生成器，并执行必要的清理逻辑。
